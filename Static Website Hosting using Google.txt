Static Website Hosting using Google Cloud Storage

Step 1: Start Lab 

Login to Google Cloud Skills Boost → Start the assigned lab → Click 'Open Google Console'.
Ensure project ID matches lab instructions. 

Step 2: Create HTML Files 
Create index.html and error.html locally using Notepad/VS Code. Save files. 

Step 3: Create Bucket 
Navigate to Navigation Menu → Cloud Storage → Buckets → Create. 
Enter unique bucket name, Region, Standard storage,
Uniform access control, then Create.

Step 4: Upload Files 
Open bucket → Upload → Upload files → Select index.html and error.html. 

Step 5: Disable Public Access Prevention 
Open bucket → Configuration/Protection → 
Edit Public Access Prevention → Uncheck 
'Enforce public access prevention' → Confirm. 

Step 6: Grant Public Access 
Open Permissions → Grant Access → Principal: 
allUsers → Role: Storage Object Viewer → Save. 

Step 7: Configure Static Website 
Go to Configuration → Static Website Hosting → Set index.html
and error.html → Save. Step 8: Access Website 
Open Objects tab → Click index.html → Copy Public URL → Paste in browser.
Troubleshooting 
• Access denied → permissions missing • XML error → website configuration missing • 404 error → wrong file name 
• Still private → public access prevention enabled • Lab expired → restart lab 
Expected Result 
Website loads publicly without login using URL.
