Practical 5: Xen Nested Virtualization Lab Manual
End-to-End Step-by-Step Guide (Until XenCenter Connection)

tep 1 — Enable Virtualization in BIOS
Restart your computer
Enter BIOS/UEFI using F2 / F10 / DEL key
Go to CPU Configuration
Enable Intel VT-x / AMD-V and Intel VT-d (if available)
Save and exit BIOS
This allows nested virtualization support for Xen installation.

Step 2 — Open VMware Workstation / VirtualBox
Launch VMware Workstation
Click Create a New Virtual Machine
Choose Custom (Advanced)
This starts Xen virtual machine setup.

Step 3 — Configure New Virtual Machine
Set VM Name: Xen-Lab
Choose Guest OS: Linux
Version: Other Linux 64-bit
Assign:
1 Processor
4 Cores
8 GB RAM
80 GB Disk
Network: Bridged
Click Finish
This creates VM for Xen hypervisor installation.

Step 4 — Enable Nested Virtualization
Right-click Xen-Lab VM
Open Settings → Processors
Enable:
Virtualize Intel VT-x/EPT or AMD-V
Virtualize IOMMU
Click OK
This is mandatory or Xen will fail to detect virtualization.

Step 5 — Attach XCP-ng ISO
Download XCP-ng ISO file
Open VM Settings → CD/DVD
Select Use ISO Image File
Browse and attach downloaded ISO
This mounts Xen installer disk into VM.

Step 6 — Start Xen Virtual Machine
Power on Xen-Lab VM
Boot from attached ISO
Xen installer screen will appear
This begins XCP-ng installation.

Step 7 — Start XCP-ng Installation
Select Install XCP-ng
Press Enter
Installer starts copying required setup files.

Step 8 — Configure Installation Settings
Choose Keyboard: English
Select entire disk for installation
Choose Installation Source: Local Media
Set secure root password
These settings prepare Xen server system files.

Step 9 — Configure Network Settings
Choose DHCP (recommended)
OR manually enter:
IP Address
Subnet Mask
Gateway
DNS Server
Set Hostname: xen-nested
Set NTP: pool.ntp.org
This enables network access to Xen server.

Step 10 — Complete Installation
Wait until installation finishes
Remove ISO when prompted
Reboot system
Xen hypervisor is now installed.

Step 11 — Verify Xen Console Screen
After reboot, Xen console appears
Note down:
Hostname
IP Address
Xen Version
Example: 192.168.1.120
This IP is required for XenCenter connection.

Step 12 — Install XenCenter on Host PC
On Windows host machine, download XenCenter
Run installer
Complete installation with default options
XenCenter is management tool for Xen servers.

Step 13 — Open XenCenter
Launch XenCenter application
Click Add Server
This starts Xen host connection process.

Step 14 — Connect to Xen Server
Enter:
Server: Xen VM IP Address
Username: root
Password: root password created earlier
Click Add
If successful, Xen host status becomes Connected.

Step 15 — Confirm Successful Setup
Verify Xen host appears in XenCenter dashboard
Xen nested virtualization lab is complete.

Final Result
Xen (XCP-ng) successfully installed inside virtual machine and connected through XenCenter.
